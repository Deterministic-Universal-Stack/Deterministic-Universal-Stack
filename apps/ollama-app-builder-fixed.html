What is Ollama App Builder?

Ollama App Builder is a local development environment for creating web applications using AI assistance. It combines the power of Ollama's local models with constraint-based validation and automatic repair capabilities.

Key Features
  • ✨ AI-powered component generation using Ollama
  • 📐 Constraint-based building (CSS, HTML, JS limits)
  • 🔧 Automatic repair and validation
  • 🎨 Multiple design styles and themes
  • 📱 Mobile-first responsive design
  • ⚡ Real-time build feedback
  • 🔄 Iteration and refinement tools
Constraints Overview

Every build must satisfy these constraints:

Valid HTML5 markup
CSS with scoped variables
Vanilla JavaScript (no external deps)
Responsive design (mobile-first)
WCAG accessibility standards
Performance optimization
File size limits enforced
Auto-repair on failures
Getting Started

1. Ensure Ollama is running on localhost:11434

2. Create a new project from the Create App page

3. Use the Builder to request components

4. Review and validate the generated code

5. Iterate using the repair and rebuild tools

API Integration

The builder uses the following endpoints:

POST /api/generate - Generate component code
POST /api/validate - Validate HTML/CSS/JS
POST /api/repair - Auto-repair failed builds
GET /api/models - List available models
Best Practices
  • Be specific in component requests
  • Use the constraints as guidelines
  • Enable auto-repair for faster iteration
  • Test responsive design on mobile
  • Validate before deploying
  • Keep components modular

Complete Build Constraints

HTML Constraints

Must be valid HTML5
DOCTYPE must be present
Must include meta charset
Must be self-contained (no external resources)
Semantic HTML structure required

CSS Constraints

Use CSS custom properties (variables)
No !important declarations
Mobile-first approach required
Max 50KB total size
Scoped styling (no global pollution)
Responsive breakpoints: 320px, 768px, 1024px

JavaScript Constraints

Vanilla JavaScript only (no frameworks)
No external library dependencies
Max 30KB total size
Must use modern ES6+ syntax
Proper error handling required

Performance Constraints

Total file size: Max 100KB
LCP: < 2 seconds
FID: < 100ms
CLS: < 0.1
Lazy load images where possible

Accessibility Constraints

WCAG 2.1 Level AA compliance
Proper heading hierarchy
Color contrast ratio ≥ 4.5:1
Keyboard navigation support
ARIA labels where appropriate

Build Process Constraints

Validation runs on every build
Failed validation triggers auto-repair
Max 3 auto-repair attempts
Human review required after 3 failures
All changes logged and version controlled
